# FitFindr - planning.md

## Tools

### Tool 1: search_listings

**What it does:**
Searches the mock secondhand listings dataset for clothing items that match a natural language description. It can also filter by size and maximum price before ranking results by keyword, category, color, and style relevance.

**Input parameters:**
- `description` (str): Natural language clothing request, such as "black jacket" or "vintage graphic tee".
- `size` (str | None): Optional requested size, such as "M", "W28", or "US 8".
- `max_price` (float | None): Optional maximum price filter.

**What it returns:**
A `list[dict]` of matching listing dictionaries sorted with the best match first. Each result can include `id`, `title`, `description`, `category`, `style_tags`, `size`, `condition`, `price`, `colors`, `brand`, and `platform`.

**What happens if it fails or returns nothing:**
The tool returns an empty list. The planning loop stops and returns: "No listings found matching your request. Try broader keywords such as 'jacket', 'outerwear', or a different color."

---

### Tool 2: suggest_outfit

**What it does:**
Generates a styling recommendation for the selected listing. If the user has wardrobe items, it tries to build the outfit from named pieces in that wardrobe; if not, it gives general styling advice.

**Input parameters:**
- `new_item` (dict): The listing selected from `search_listings`.
- `wardrobe` (dict): A wardrobe dictionary with an `items` list.

**What it returns:**
A non-empty `str` containing one outfit recommendation.

**What happens if it fails or returns nothing:**
If `new_item` is missing or invalid, the tool returns an empty string. The planning loop stops and returns: "Unable to generate an outfit recommendation for this item."

---

### Tool 3: create_fit_card

**What it does:**
Creates the final recommendation card shown to the user. It includes the selected item, category, price, platform, outfit suggestion, and a short explanation of why the outfit works.

**Input parameters:**
- `outfit` (str): Recommendation generated by `suggest_outfit`.
- `new_item` (dict): The selected listing.

**What it returns:**
A formatted `str` fit card.

**What happens if it fails or returns nothing:**
If the outfit text or selected item is missing, the tool returns: "Unable to create the final style card. Please try again." The planning loop stops with the same error.

---

### Additional Tools

No additional agent tools are required. The implementation uses helper functions for parsing the query, scoring listing matches, choosing wardrobe pieces, and formatting prices.

## Planning Loop

**How does your agent decide which tool to call next?**
The agent follows a fixed sequence because each step depends on the previous one:

1. Receive the user query.
2. Parse the query into `description`, `size`, and `max_price`.
3. Call `search_listings(description, size, max_price)`.
4. If no results are found, store an error and stop.
5. Select the first result as `session["selected_item"]`.
6. Call `suggest_outfit(selected_item, wardrobe)`.
7. If no outfit is generated, store an error and stop.
8. Call `create_fit_card(outfit_suggestion, selected_item)`.
9. If the card cannot be created, store an error and stop.
10. Return the completed session.

The loop knows it is done when `session["fit_card"]` has been created or when one of the error branches sets `session["error"]`.

---

## State Management

**How does information from one tool get passed to the next?**
The agent keeps one session dictionary for the entire interaction:

```python
session = {
    "query": query,
    "parsed": {},
    "search_results": [],
    "selected_item": None,
    "wardrobe": wardrobe,
    "outfit_suggestion": None,
    "fit_card": None,
    "error": None,
}
```

The parsed query is stored in `session["parsed"]`. Search results are stored in `session["search_results"]`, and the first result is stored in `session["selected_item"]`. That selected item is passed into `suggest_outfit`, whose output is stored in `session["outfit_suggestion"]`. Finally, the selected item and outfit suggestion are passed into `create_fit_card`, and the result is stored in `session["fit_card"]`.

---

## Error Handling

| Tool | Failure mode | Agent response |
|------|-------------|----------------|
| search_listings | No results match the query | Stop execution and return "No listings found matching your request. Try broader keywords such as 'jacket', 'outerwear', or a different color." |
| suggest_outfit | Selected item is missing or no recommendation is generated | Stop execution and return "Unable to generate an outfit recommendation for this item." |
| create_fit_card | Outfit input or selected item is missing | Stop execution and return "Unable to create the final style card. Please try again." |

## A Complete Interaction (Step by Step)

**Example user query:** "Find me a black jacket under $80 in size M."

**Step 1:**
The agent parses the query:

```python
{
    "description": "black jacket",
    "size": "M",
    "max_price": 80.0,
}
```

**Step 2:**
The agent calls:

```python
search_listings("black jacket", size="M", max_price=80.0)
```

The tool filters listings by price and size, scores the remaining listings, and returns the best matches.

**Step 3:**
The agent selects the first result:

```python
session["selected_item"] = results[0]
```

**Step 4:**
The agent calls:

```python
suggest_outfit(session["selected_item"], wardrobe)
```

The tool returns a styling suggestion using the selected item and either the example wardrobe or general styling advice.

**Step 5:**
The agent stores:

```python
session["outfit_suggestion"] = outfit
```

**Step 6:**
The agent calls:

```python
create_fit_card(session["outfit_suggestion"], session["selected_item"])
```

**Final output to user:**
The user sees three panels in the Gradio app: the selected listing, the outfit recommendation, and a formatted fit card.
